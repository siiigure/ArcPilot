# ArcPilot 前端 2.0 设计方案（定稿）

**方案名称**：流体中心布局（Fluid-Center UI）  
**定位**：在不变更路由与页面上下文的前提下，将主壳从「固定像素三栏」升级为「全宽权重分配 + 可拖拽主次关系」的工作台式体验。  
**技术基线**：现有栈为 React 19、Vite、TanStack Router、Tailwind CSS 4、Radix UI；本方案新增 **`react-resizable-panels`** 作为分栏实现。

---

## 1. 设计目标

| 目标 | 说明 |
|------|------|
| 主次可塑 | 论坛与 AI 同屏协作时，用户能按任务在「并重 / 论坛沉浸 / AI 深度」间切换。 |
| 单一状态源 | 预设比例、拖拽结果、持久化字段同源管理，避免「按钮一套、拖拽一套」冲突。 |
| 内容优先 | 壳层尽量隐形；复杂内容（长文、图纸、代码）不被炫目装饰抢戏。 |
| 可访问性 | 分割线可聚焦、键盘可调；不依赖唯一 hover 通道传达关键状态。 |
| 跨断点不丢态 | 窄屏切换布局形态时，论坛与 AI 的核心状态挂在全局层，不因卸载壳子而丢失。 |

---

## 2. 核心布局模型

不把页面理解为「左中右三块固定宽」，而理解为 **100% 宽度的水平容器**，在 **Nav + 主工作区** 内分配空间：

- **左侧导航（Nav）**  
  - 标准宽度：**200px**（固定或可折叠为 **64px** 图标栏，与「AI 深度」场景一致）。  
  - 与中间 + 右侧 **不** 放在同一 `PanelGroup` 内（避免与「仅调论坛/AI 比例」的心智混淆）；Nav 单独一列，右侧为 **论坛 | AI** 的 `PanelGroup`。

- **主工作区（Forum + AI）**  
  - 由 `PanelGroup direction="horizontal"` 管理，两列均为 **flex 权重**（百分比），非固定像素。

### 2.1 三种预设（用户心智）

| 预设 | Nav | 论坛 | AI | 场景 |
|------|-----|------|-----|------|
| **均衡** | 200px | ≈50% | ≈50% | 边刷论坛边用 AI。 |
| **论坛主导** | 200px | ≈80% | ≈20% | 读长文、找灵感；AI 窄条待命。 |
| **AI 主导** | 64px（极简） | ≈20%（目录态） | ≈80% | 深度生成；论坛作素材列表。 |

预设仅表示 **默认或一键恢复的比例**；用户拖拽后的结果记为 **自定义布局**，与预设按钮通过统一 API（如 `setLayout([forumPct, aiPct])`）更新。

### 2.2 与现有实现的差异

当前壳层使用 `max-w-[1320px]` 与中间 `max-w-[680px]` 的「居中阅读盒」。**2.0 定稿：外壳改为全宽流体（`w-full`）**；若需限制行宽，仅在 **论坛正文区域**（如文章 `Article` 容器）上使用 `max-w-*`，**不**限制整块 Panel，以便侧栏与 AI 贴边利用宽屏。

## 3. 交互设计

### 3.1 中轴拖拽（论坛 / AI 分界）

- **命中区**：建议 **4px** 宽（视觉上可用内嵌细线 + 扩大可点区域实现）。  
- **悬停**：可用品牌色强调；**必须**配合 **焦点环**（键盘聚焦时可见），避免仅靠 hover。  
- **拖拽中**：对易被抢事件的区域（如 **iframe、地图、WebGL、复杂图表**）可对内容区临时 `pointer-events: none` 或透明遮罩，减少拖拽卡顿与误触。  
- **双击**：在 **均衡（如 50:50）** 与 **上一次用户自定义比例** 之间切换；双击需防抖，避免误触。「上次偏好」= 最近一次通过拖拽确认的布局（见持久化）。  

### 3.2 顶部区域

- 搜索栏适度缩小；**中部**展示当前上下文状态（例：`论坛: 遥感影像` / `AI: 助手`）。  
- **右上角三个图标**：对应「论坛主导 / 均衡 / AI 主导」，与拖拽共用同一套 `setLayout`，不做第二套比例逻辑。

### 3.3 主体提示（视觉定稿：克制）

- **不采用**大面积、高饱和品牌色（如 #A3D139）动态阴影作为「谁是主体」的主手段——易干扰图纸、代码与长文阅读。  
- **推荐**：  
  - 用 **极弱** 的明暗对比区分主次（如主体 `bg-background`、次主体 `bg-muted/30`）；或  
  - 中轴 **1px** 冷色微光 / 分隔线强化；  
  - 预设按钮的 **选中态** 作为不依赖 hover 的焦点指示。

## 4. 论坛内容适配（Content Adaptation）

当 **论坛列**过窄时，卡片信息密度过高，需降级为「列表态」：

- **触发条件（定稿）**：以 **论坛 Panel 的像素宽度** 为准（通过 `ResizeObserver` 或 `react-resizable-panels` 提供的尺寸 API 获取），**禁止**使用未定义的 `width` 变量。  
- **建议阈值**：与产品实测绑定；初值可采用 **约 30% 相对宽度** 或 **绝对像素阈值**（二选一写死，文档中记录最终数值）。  
- **滞回（Hysteresis）**：避免在阈值附近抖动。示例思路：进入 compact 与退出 full 使用不同阈值（如 `< 280px` 进入，`> 320px` 退出，数值可再调）。  
- **compact 行为**：隐藏发帖人详情、复杂点赞区、大图预览；卡片收敛为 **单行标题列表**，保证可点击跳转，主视线留在 AI。  
- **性能**：仅在 `viewMode` 跨越阈值时更新状态，避免拖拽每一帧驱动大列表重渲染。

---

## 5. AI 区域：对话与 Artifacts

- **非主体**：单栏对话流为主。  
- **主体**（AI 列占比 **≥ 约 60%**，具体比例可配置）：**左侧对话 + 右侧预览**（Artifacts：文档 / 代码 / 图等）。  
- **联动**：当消息中产生可预览块时，右侧自动呈现；可选增强：对话中点击某段说明 ↔ 预览区对应块高亮（下阶段可迭代）。  
- **重型内容**（遥感图、WebGL、高清底图）：  
  - 拖拽过程中除 `pointer-events` 处理外，对预览区宽度变化使用 **防抖**；  
  - 可选：**拖拽中**用低清占位 / 静态截图，**拖拽结束**（`onLayout` 稳定后延迟约 **200ms**）再恢复高清渲染，避免连续 reflow 卡死主线程。

## 6. 响应式与窄屏

| 断点 | 策略 |
|------|------|
| **Desktop（建议 `lg+`）** | 完整三栏：Nav + `PanelGroup(论坛, AI)`，可拖拽。 |
| **Tablet（`md`–`lg`）** | Nav 收缩为图标栏；论坛与 AI 仍可 `PanelGroup`（比例策略可另定）。 |
| **Mobile（`<md`）** | **不**在窄屏强行使用桌面 `PanelGroup`；改为 **单主视图 + AI 抽屉 / 底部 Sheet / FAB 切换**。 |

**状态与生命周期（定稿）**：  
论坛列表与 AI 会话等 **核心业务状态** 必须挂在 **Layout 之上的 Context / 全局 Store**（与具体状态库无关，原则是提升层级）。窄屏仅更换 **容器与呈现方式**（如 Sheet 包裹同一 `AIChat` 子树），避免因卸载桌面壳导致 **未提升的局部 state 丢失**。若技术上必须切换树结构，须保证同一 store 承接，做到「换壳不换脑」。

## 7. 布局状态、持久化与首屏

### 7.1 单一真相

- 字段建议：`preset: 'forum' | 'balanced' | 'ai' | 'custom'`、`sizes: [forum, ai]`（百分比）、可选 `navCollapsed: boolean`。  
- 所有入口（预设按钮、拖拽结束、双击恢复）只更新这一处，再驱动 `PanelGroup`。

### 7.2 持久化与 Hydration（定稿）

- **纯 localStorage**：服务端/Vite SPA 首屏 HTML **无法**读取，若首屏就按存储值渲染 Panel，易出现 **闪烁** 或与 SSR 不同源的 mismatch（若未来引入 SSR）。  
- **定稿策略（二选一或组合）**：  
  1. **Cookie 存布局**（需控制体积与安全），服务端若将来 SSR 可读，首屏一致；或  
  2. **首屏固定默认布局**，客户端 `useEffect` 后再 `setLayout`；过渡用 **短骨架 / 淡入** 掩盖跳跃（接受极短布局动画优于硬跳）。  

不在文档中承诺「仅靠 defaultLayout + localStorage 即可无闪烁 SSR」，除非已选 Cookie 或同构读存储方案。

---

## 8. 无障碍与键盘

- `PanelResizeHandle`：**可聚焦**；**左右方向键**按步进调整（建议 **5%** 步长，可配置）。  
- 为分割线提供合适 **aria** 属性（角色/标签由组件库文档与 a11y 审计确定）。  
- 关键状态（当前预设、谁是主工作区）在顶栏或按钮上有 **非 hover** 可见反馈。

---

## 9. 路由与例外页

- **默认**：`Outlet` 仍在 **论坛 Panel** 内，**路由不变**，操作在同一上下文完成。  
- **例外**：设置页、全屏编辑器等可在路由 meta 声明 **隐藏右栏 / 单栏布局**，避免机械 50:50。

## 10. 技术实现要点（示例级）

依赖：`react-resizable-panels`。

结构示意（Nav 在 `PanelGroup` 外）：

```tsx
<div className="flex h-svh w-full min-h-0 bg-background">
  <nav className="w-[200px] shrink-0 border-r ...">
    <NavigationContent />
  </nav>
  <PanelGroup direction="horizontal" className="min-h-0 flex-1" onLayout={persistSizes}>
    <Panel defaultSize={50} minSize={20}>
      <section className="h-full min-h-0 overflow-y-auto">
        <ForumList viewMode={forumViewMode} />
      </section>
    </Panel>
    <PanelResizeHandle className="w-1 ..." />
    <Panel defaultSize={50} minSize={20}>
      <section className="h-full min-h-0 ...">
        <AIChat layoutVariant={aiLayoutVariant} />
      </section>
    </Panel>
  </PanelGroup>
</div>
```

`forumViewMode` 由 **论坛列宽度 + 滞回** 推导；`aiLayoutVariant` 由 **AI 列占比阈值** 推导。

## 11. 验收标准与落地任务

| 序号 | 任务 | 验收要点 |
|------|------|----------|
| 1 | 定稿 compact 阈值 | 实测卡片拥挤临界点，写入本文档与常量。 |

| 3 | 统一布局状态 + 持久化 | 预设与拖拽一致；刷新后恢复；首屏无剧烈跳变（骨架或 Cookie 方案）。 |
| 4 | Handle 键盘 | 焦点可见；方向键调节比例。 |
| 5 | 窄屏 | `<md` 抽屉/Sheet + 状态不丢。 |
| 6 | 性能 | 100+ 帖子列表拖拽流畅；重型预览拖拽策略生效。 |
| 7 | a11y / E2E | Playwright 覆盖：拖拽、预设、键盘、窄屏切换关键路径。 |
