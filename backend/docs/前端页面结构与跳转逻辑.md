# ArcPilot 前端页面结构与跳转逻辑方案

本文档覆盖核心业务场景下的**前端页面结构图**与**跳转逻辑**：系统级导航拓扑、各页面功能/布局/入口/参数/下级路径，以及关键路由参数表。与 [数据库逻辑模型](./数据库逻辑模型.md) 中的 User、Invite、Post、Reply、Like、Tag、AiSession、AiMessage、AiAnswer、AiCitation 等实体对应。

---

## 一、系统级导航拓扑图描述

### 1.1 拓扑总览（文字 + 层级）

```
                        ┌─────────────────────────────────────────────────────────────────┐
                        │                        全局入口 / 壳层                              │
                        │  Header(Logo/搜索/主题/通知/Add question/用户) + 可选 LeftSidebar   │
                        └─────────────────────────────────────────────────────────────────┘
                                                      │
         ┌─────────────────────────────────────────────┼─────────────────────────────────────────────┐
         │                                             │                                             │
         ▼                                             ▼                                             ▼
┌─────────────────┐                         ┌─────────────────┐                         ┌─────────────────┐
│   认证与邀请     │                         │   社区 (Feed)    │                         │   AI 助手       │
│  /login         │                         │  /              │                         │  /ai            │
│  /register      │◄───────────────────────►│  /topic/[slug]  │                         │  /ai/[sessionId]│
│  /invite/[code] │                         │  /post/[id]      │                         │  /ai/new        │
│  /forgot-password│                        │  /search         │                         └────────┬────────┘
└────────┬────────┘                         │  /post/new       │                                  │
         │                                  │  /post/[id]/edit │                                  │
         │ 登录/注册成功                     └────────┬────────┘                                  │ 引用闭环
         │ 统一跳转 / 或 /ai                          │                                            │
         │                                           │ 帖子/标签/用户/搜索                          ▼
         │                                           ▼                                    ┌─────────────────┐
         │                                  ┌─────────────────┐                            │  引用与资源     │
         └────────────────────────────────►│   用户与设置     │                            │  /citation/[id] │
                                            │  /profile/[uid] │                            │  (预览/下载)    │
                                            │  /settings      │                            └─────────────────┘
                                            │  /notifications │
                                            └─────────────────┘
```

- **三条主干线**：认证与邀请、社区、AI 助手；用户与设置作为横切（从 Header/侧栏/帖子作者等多处进入）。
- **闭环**：邀请链接 → 注册 → 登录 → 首页/AI；搜索 → 结果 → 帖子/用户/标签 → 返回或再搜索；帖子详情 → 回复/点赞 → 留在本页；AI 会话 → 答案引用 → 引用详情/下载 → 返回会话。

### 1.2 与数据模型的对应关系

| 模块       | 主要实体               | 典型路由 |
|------------|------------------------|----------|
| 认证与邀请 | User, Invite           | `/login`, `/register`, `/invite/[code]` |
| 社区       | Post, Reply, Like, Tag | `/`, `/post/[id]`, `/topic/[slug]`, `/search` |
| 用户与设置 | User (profile_metadata)| `/profile/[uid]`, `/settings` |
| AI 助手    | AiSession, AiMessage, AiAnswer, AiCitation | `/ai`, `/ai/[sessionId]`, `/ai/new` |
| 引用与资源 | AiCitation (url / object_storage) | `/citation/[id]`（或弹窗/新标签） |

## 二、按页面逐一展开

以下每个页面按：**功能与布局**、**入口来源**、**携带参数**、**下级路径**、**涉及的闭环** 五项说明。

---

### 2.1 认证与邀请

#### 2.1.1 登录页 `/login`

| 项         | 说明 |
|------------|------|
| **功能与布局** | 登录表单（邮箱 + 密码）、「忘记密码」链接、「去注册」链接；可选第三方登录占位。 |
| **入口来源**   | 未登录访问需登录页时重定向、Header「用户」点击未登录时、从 `/register` 或 `/invite/[code]` 完成注册后跳转至此再登录（若未实现自动登录）。 |
| **携带参数**   | 可选 `?redirect=/ai` 或 `?redirect=/post/123`，登录成功后跳转；无则默认 `/`。 |
| **下级路径**   | 成功 → `redirect` 或 `/`；「去注册」→ `/register`；「忘记密码」→ `/forgot-password`。 |
| **闭环**       | 邀请闭环：`/invite/[code]` → 注册 → 本页登录 → 首页/AI。 |

#### 2.1.2 注册页 `/register`

| 项         | 说明 |
|------------|------|
| **功能与布局** | 注册表单（邮箱、密码、展示名等）；可选「使用邀请码」输入框或通过 `/invite/[code]` 预填。 |
| **入口来源**   | 未登录时 Header/弹窗「注册」、邀请页「使用此邀请码注册」跳转、登录页「去注册」。 |
| **携带参数**   | 无路径参数；可选 query `?invite=CODE` 预填邀请码。 |
| **下级路径**   | 成功 → `/login?redirect=/` 或直接登录并跳转 `/`/`/ai`；「去登录」→ `/login`；带邀请码时可 → `/invite/[code]` 展示邀请信息再回本页。 |
| **闭环**       | 邀请闭环起点之一（用户从邀请链接来后可在此完成注册）。 |

#### 2.1.3 邀请注册页 `/invite/[code]`

| 项         | 说明 |
|------------|------|
| **功能与布局** | 展示邀请信息（创建者、有效期、剩余次数等）；CTA「使用邀请码注册」跳转注册并携带 code。 |
| **入口来源**   | 仅外部分享链接：`/invite/{code}`（邮件/消息等）。 |
| **携带参数**   | 路径参数：`code`（Invite.code）。 |
| **下级路径**   | 「去注册」→ `/register?invite={code}` 或带 state 的注册流；已登录可提示「您已登录」并 → `/`。 |
| **闭环**       | 邀请闭环：本页 → 注册 → 登录 → 首页/AI；后端校验 code 状态（pending/used/expired）。 |

#### 2.1.4 忘记密码 `/forgot-password`

| 项         | 说明 |
|------------|------|
| **功能与布局** | 输入邮箱，提交后提示「已发送重置链接」；可选「返回登录」。 |
| **入口来源**   | 登录页「忘记密码」链接。 |
| **携带参数**   | 无。 |
| **下级路径**   | 提交后留在本页或跳转 `/login`；「返回登录」→ `/login`。 |
| **闭环**       | 无独立闭环，属于认证支线。 |

---

### 2.2 社区（Feed 与内容）

#### 2.2.1 首页 Feed `/`

| 项         | 说明 |
|------------|------|
| **功能与布局** | 三栏：LeftSidebar（Home/Following/Answer/Notifications + Spaces/Topics）、主区帖子流（PostCard 列表）、RightSidebar（Arcpilot AI 快捷入口或嵌入）。Header 全站统一。**MVP 范围**：Following 为占位（当前数据模型无关注关系，后续迭代）；Spaces 即 Topics，对应 Tag 聚合页。 |
| **入口来源**   | 根路径访问、登录/注册成功后默认、Logo 点击、左侧「Home」。 |
| **携带参数**   | 可选 query：`?tag=slug`、`?sort=latest|popular`、分页 `?page=1`。 |
| **下级路径**   | 帖子卡片点击 → `/post/[id]`；Topic 点击 → `/topic/[slug]`；Header「Add question」→ `/post/new`；搜索提交 → `/search?q=...`；右侧 AI → `/ai` 或 `/ai/new`。 |
| **闭环**       | 从帖子详情、标签页、搜索返回本页；从 AI 会话返回。 |

#### 2.2.2 标签/空间页 `/topic/[slug]`

| 项         | 说明 |
|------------|------|
| **功能与布局** | 与首页同三栏；主区标题为 Tag 名称，列表为该 Tag 下的 Post（与数据模型 Tag slug 对应）。 |
| **入口来源**   | LeftSidebar 的 Spaces/Topics 链接、Feed 中帖子标签点击、搜索结果为标签时。 |
| **携带参数**   | 路径：`slug`（Tag.slug）；可选 `?sort=...`、`?page=...`。 |
| **下级路径**   | 同首页：帖子 → `/post/[id]`、「Add question」→ `/post/new`、搜索 → `/search`、AI → `/ai`。 |
| **闭环**       | 从帖子详情或搜索返回本页。 |

#### 2.2.3 帖子详情页 `/post/[id]`

| 项         | 说明 |
|------------|------|
| **功能与布局** | 主区：帖子正文（Post.body）、作者信息、Tag 列表、点赞/评论数；下方 Reply 列表（可分页）；回复框（已登录）。侧栏可保留或收缩。 |
| **入口来源**   | Feed/PostCard 点击、标签页帖子、搜索结果、通知「有人回复了你的帖子」、个人主页「我的帖子」。 |
| **携带参数**   | 路径：`id`（Post.id）；可选 `?highlight=replyId` 高亮某条回复、`?comment=1` 滚动到评论区。 |
| **下级路径**   | 作者点击 → `/profile/[uid]`；Tag 点击 → `/topic/[slug]`；「编辑」（本人）→ `/post/[id]/edit`；回复提交留在本页（或滚动到新回复）。 |
| **闭环**       | 回复/点赞闭环：本页内完成，无路由跳转；编辑后返回本页。 |

#### 2.2.4 发帖页 `/post/new`

| 项         | 说明 |
|------------|------|
| **功能与布局** | 表单：标题、正文、可选 Tag 多选、可选 project_geom 上传/绘制（与 Post.project_geom 对应）。提交前可预览。 |
| **入口来源**   | Header「Add question」、左侧「Answer」或快捷入口。 |
| **携带参数**   | 可选 `?topic=slug` 预选 Tag。 |
| **下级路径**   | 提交成功 → `/post/[id]`（新帖）；取消/返回 → `/` 或上一页。 |
| **闭环**       | 发帖闭环：本页 → 帖子详情。 |

#### 2.2.5 帖子编辑页 `/post/[id]/edit`

| 项         | 说明 |
|------------|------|
| **功能与布局** | 与发帖表单类似，预填当前 Post 数据；仅作者或管理员可见。 |
| **入口来源**   | 帖子详情页「编辑」按钮。 |
| **携带参数**   | 路径：`id`（Post.id）。 |
| **下级路径**   | 保存 → `/post/[id]`；取消 → `/post/[id]`。 |
| **闭环**       | 编辑闭环：详情 → 编辑 → 详情。 |

#### 2.2.6 搜索页 `/search`

| 项         | 说明 |
|------------|------|
| **功能与布局** | Header 搜索框提交后进入；主区为搜索结果：Tab 或分区（帖子 / 用户 / 标签），列表 + 分页。 |
| **入口来源**   | Header 搜索表单提交（含 use-search 的 handleSearch）。 |
| **携带参数**   | Query：`q`（关键词）、可选 `type=post|user|tag`、`page=1`。 |
| **下级路径**   | 帖子 → `/post/[id]`；用户 → `/profile/[uid]`；标签 → `/topic/[slug]`；修改搜索留在本页并更新 `q`。 |
| **闭环**       | 搜索闭环：搜索 → 结果 → 进入详情 → 返回或再搜。 |

### 2.3 用户与设置

#### 2.3.1 个人主页 `/profile/[uid]`

| 项         | 说明 |
|------------|------|
| **功能与布局** | 用户展示（display_name、avatar_url、bio）；Tab：帖子/回复/点赞；列表与 Feed 卡片一致。与 User 及 Post/Reply/Like 对应。 |
| **入口来源**   | 帖子/回复作者点击、搜索用户结果、Header 用户菜单「我的主页」、通知「有人关注/回复」等。 |
| **携带参数**   | 路径：`uid`（User.id）；可选 `?tab=posts|replies|likes`、`?page=...`。 |
| **下级路径**   | 帖子 → `/post/[id]`；「设置」（本人）→ `/settings`；关注/取关（占位，当前无 Follow 表，后续迭代）留在本页或刷新。 |
| **闭环**       | 从帖子/搜索进入再返回。 |

#### 2.3.2 设置页 `/settings`

| 项         | 说明 |
|------------|------|
| **功能与布局** | 账户信息（展示名、头像、邮箱）、隐私、通知；**环境预设默认**（profile_metadata：industry、region 等，用于生成 AiSession.environment_preset）。 |
| **入口来源**   | Header 用户下拉「设置」、个人主页「设置」。 |
| **携带参数**   | 无或 `?tab=account|privacy|environment`。 |
| **下级路径**   | 保存后留在本页或 → `/profile/[uid]`；返回 → 上一页或 `/`。 |
| **闭环**       | 设置闭环：保存后影响新建 AI 会话的默认环境。 |

#### 2.3.3 通知中心 `/notifications`

| 项         | 说明 |
|------------|------|
| **功能与布局** | 列表：回复、点赞、@、系统通知等；可未读/已读、分页。左侧可保留精简导航。 |
| **入口来源**   | Header 铃铛、左侧「Notifications」。 |
| **携带参数**   | 可选 `?page=...`、`?filter=all|unread`。 |
| **下级路径**   | 点击某条 → 对应帖子 `/post/[id]` 或回复高亮、或用户 `/profile/[uid]`。 |
| **闭环**       | 通知 → 帖子/用户 → 返回通知或 Feed。 |

### 2.4 AI 助手

#### 2.4.1 AI 会话列表 `/ai`

| 项         | 说明 |
|------------|------|
| **功能与布局** | 左侧或主区：AiSession 列表（标题、摘要、时间）；右侧可为空或占位「选一个会话或新建」。Header 可保留。 |
| **入口来源**   | 右侧边栏「Arcpilot AI」、Header「Add question」旁可增加「AI」、首页/任意页导航入口。 |
| **携带参数**   | 可选 `?page=...`。 |
| **下级路径**   | 点击会话 → `/ai/[sessionId]`；「新建会话」→ `/ai/new` 或直接创建后 → `/ai/[sessionId]`。 |
| **闭环**       | 从单会话页返回列表；引用详情返回会话再返回列表。 |

#### 2.4.2 单会话对话页 `/ai/[sessionId]`

| 项         | 说明 |
|------------|------|
| **功能与布局** | 主区：AiMessage 流（user/assistant），支持 environment_preset 展示与 instant_override 的体现（如版本变更提示）；输入框可支持「即时修正」文案或结构化输入。右侧可展示当前有效 metadata、引用列表。每条 assistant 消息下可展开 AiAnswer 的 AiCitation 列表。 |
| **入口来源**   | 会话列表点击、新建会话成功后、通知/邮件「继续对话」链接。 |
| **携带参数**   | 路径：`sessionId`（AiSession.id）；可选 `?message=messageId` 滚动到某条、`?focus=answer` 聚焦最新答案。 |
| **下级路径**   | 引用点击 → `/citation/[id]` 或弹窗/新标签预览与下载；「新建会话」→ `/ai/new`；返回列表 → `/ai`。 |
| **闭环**       | **引用闭环**：本页 → 引用详情/下载 → 返回本页或新标签关闭后继续对话。 |

#### 2.4.3 新建 AI 会话 `/ai/new`

| 项         | 说明 |
|------------|------|
| **功能与布局** | 可选：环境预设表单（arcgis_pro_version、project_crs、region、industry 等，对应 AiSession.environment_preset）；或简化为「直接开始」则用 User.profile_metadata 默认。提交即创建 AiSession 并跳转对话页。 |
| **入口来源**   | 会话列表「新建」、Header/侧栏「新对话」。 |
| **携带参数**   | 可选 `?preset=id` 从模板带出预设。 |
| **下级路径**   | 创建成功 → `/ai/[sessionId]`；取消 → `/ai`。 |
| **闭环**       | 新建 → 对话页，构成 AI 使用闭环起点。 |

---

### 2.5 引用与资源

#### 2.5.1 引用详情/预览 `/citation/[id]`

| 项         | 说明 |
|------------|------|
| **功能与布局** | 展示 AiCitation：title、standard_code、region、source_type；根据 source_kind 展示外链（url）或对象存储（生成预签名 URL 供预览/下载）。可选 PDF 预览或下载按钮。 |
| **入口来源**   | AI 会话中某条答案的引用列表点击。 |
| **携带参数**   | 路径：`id`（AiCitation.id）；可选 `?download=1` 直接触发下载。 |
| **下级路径**   | 返回 → `/ai/[sessionId]`（若从会话来）；外链在新标签打开则无路由返回。 |
| **闭环**       | **引用闭环**：会话 → 本页（或新标签）→ 关闭/返回会话。 |

### 3.1 受保护路由与错误页

- **需登录后访问**（未登录统一重定向到 `/login?redirect=当前 URL`）：`/post/new`、`/post/[id]/edit`、`/ai`、`/ai/new`、`/ai/[sessionId]`、`/settings`、`/notifications`。发帖详情 `/post/[id]`、首页 `/`、搜索 `/search`、标签 `/topic/[slug]`、个人主页 `/profile/[uid]` 可对未登录只读开放（发帖/回复/点赞时再要求登录）。
- **错误页**：建议提供 404 页；访问他人「编辑帖」或无权限资源时返回 403 或重定向到帖子详情/首页。

## 四、闭环汇总

| 闭环名称 | 路径序列 | 说明 |
|----------|----------|------|
| **邀请闭环** | `/invite/[code]` → `/register` → `/login` → `/` 或 `/ai` | 邀请链接到落地使用 |
| **发帖闭环** | `/post/new` → `/post/[id]` | 发帖后进入详情 |
| **编辑闭环** | `/post/[id]` → `/post/[id]/edit` → `/post/[id]` | 帖子编辑来回 |
| **回复/点赞闭环** | 在 `/post/[id]` 内完成，无路由变化 | 本页内交互 |
| **搜索闭环** | `/search?q=...` → 帖子/用户/标签详情 → 返回或再搜 | 搜索到消费再搜 |
| **引用闭环** | `/ai/[sessionId]` → `/citation/[id]`（或新标签）→ 返回会话 | AI 答案引用查看/下载后回对话 |
| **新建会话闭环** | `/ai/new` → `/ai/[sessionId]` | 新建到对话 |

## 五、与现有前端的对应关系

- **当前已有**：`/`（首页 Feed）、Header、LeftSidebar（含 Topics 链接 `/topic/[slug]`，与数据模型 Tag.slug 一致）、RightSidebar（嵌入式 AI，无独立路由）。  
- **本文档为规划**：上述所有路由与参数为**目标结构**；实现时可先做核心路由（`/`、`/post/[id]`、`/ai`、`/ai/[sessionId]`、`/search`、`/login`、`/register`），再逐步补全标签页、个人主页、设置、通知、引用页等。
